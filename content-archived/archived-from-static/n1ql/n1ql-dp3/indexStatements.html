INDEX statements

The N1QL provides statements to create and drop indexes.

  • CREATE INDEX statement
    The CREATE INDEX statement creates an index on fields and nested paths. The index provides an optimized access path for N1QL queries.
  • CREATE PRIMARY INDEX statement
    This statement is used to ensure an optimized primary key index. If the primary key index already exists, the statement recognizes it and does not create a duplicate index.
  • DROP INDEX statement
    This statement removes a named index in the given bucket.
